App Design Documentation

1. Onboarding Screens

Overlay 1: Welcome Screen

  • Title: "Welcome to [App Name]"
  • Subtitle: Brief description of the app's purpose ("Control Your Phone's Wireless Radiation").
  • Call to Action (CTA) Button: "Get Started"

Overlay 2: Permissions Screen

  • Iconography: Bluetooth, WiFi, Mobile Data icons.
  • Text: "We need your permission to manage your phone’s wireless settings."
  • CTA Button: "Grant Permissions"

Overlay 3: Quick Tour

  • Interactive Area: Swipe through 3–5 screens explaining key features (e.g., "Schedule Wireless Control", "Inactivity Timer", "Safety Mode").
  • Progress Dots: At the bottom to indicate how many screens are left.
  • CTA Button: "Finish Setup"

2. Home Screen / Dashboard

Overlay 4: Home Screen Layout

  • Header: "Dashboard" with a small gear icon for Settings.
  • Quick Toggles:
    • WiFi: Toggle with "On/Off" text.
    • Bluetooth: Toggle with "On/Off" text.
    • Mobile Data: Toggle with "On/Off" text.
  • Status Indicators: Below each toggle, small text showing current status (e.g., "WiFi Connected", "Bluetooth Off").
  • Shortcut Buttons: Buttons leading to "Scheduled Controls", "Safety Mode", and "Profiles".
  • Footer: Navigation bar with icons for Home, Settings, and Profiles.

3. Settings Screen

Overlay 5: Settings Layout

  • Header: "Settings"
  • Sections:
    • Scheduled Controls: Option to set times for automatic wireless disabling.
    • Inactivity Timer: Sliders or time input for setting inactivity thresholds.
    • Safety Mode: Toggle with options to configure default behavior.
    • Advanced Settings: Link to more granular settings like individual app permissions or background activity.
  • Save Button: Fixed at the bottom to save changes.

4. Scheduled Controls Screen

Overlay 6: Scheduled Controls

  • Header: "Scheduled Controls"
  • Schedule List: List view of all existing schedules, with icons showing which are active (e.g., "Night Mode: 11:00 PM - 7:00 AM").
  • Add New Schedule: Button at the bottom with a "+" icon.
  • Edit Schedule: Tap on any schedule to open its configuration.

Overlay 7: Schedule Configuration

  • Header: "Edit Schedule"
  • Time Picker: Start time and end time inputs.
  • Days of the Week: Horizontal list of days with checkmarks for active days.
  • Connections to Disable: Toggles for WiFi, Bluetooth, Mobile Data.
  • Save/Cancel Buttons: Fixed at the bottom.

5. Inactivity Timer Screen

Overlay 8: Inactivity Timer

  • Header: "Inactivity Timer"
  • Explanation Text: "Automatically disable connections after X minutes of inactivity."
  • Timer Slider: Slider to set the duration (e.g., from 5 to 60 minutes).
  • Connections to Disable: Toggles for WiFi, Bluetooth, Mobile Data.
  • Save Button: Fixed at the bottom.

6. Safety Mode Screen

Overlay 9: Safety Mode

  • Header: "Safety Mode"
  • Toggle: Central toggle to turn Safety Mode on/off.
  • Description: "Automatically turn off all connections unless manually overridden."
  • Override Settings: Button leading to options where users can specify exceptions (e.g., "Keep WiFi on during Safety Mode").
  • Save Button: Fixed at the bottom.

7. Profile Management Screen

Overlay 10: Profile Management

  • Header: "Profiles"
  • Profile List: List of saved profiles (e.g., "Work Mode", "Sleep Mode").
  • Add New Profile: Button with a "+" icon.
  • Edit Profile: Tap on a profile to modify its settings.

Overlay 11: Profile Configuration

  • Header: "Edit Profile"
  • Profile Name Input: Text field to name the profile.
  • Profile Settings: Toggles and settings similar to the main Settings screen, but specific to this profile.
  • Set as Default: Checkbox to mark this profile as the default.
  • Save/Cancel Buttons: Fixed at the bottom.

8. Notifications and Alerts

Overlay 12: Notification Settings

  • Header: "Notification Settings"
  • List of Notifications: Toggle switches for different types of notifications (e.g., "Inactivity Timer Activated", "Safety Mode Enabled").
  • Alert Styles: Options to choose how notifications are displayed (e.g., banner, silent, etc.).
  • Save Button: Fixed at the bottom.

9. Accessibility Settings

Overlay 13: Accessibility

  • Header: "Accessibility Settings"
  • Text Size: Slider to adjust text size.
  • Color Contrast: Toggle for high contrast mode.
  • Voice Commands: Toggle to enable/disable voice control.
  • Save Button: Fixed at the bottom.

10. Responsive Design Considerations

Overlay 14: Tablet Layout

  • Ensure that all screens scale and adjust when viewed on larger devices like tablets.
  • Two-Column Layout: For larger screens, consider using a two-column layout where the settings or list views are on the left, and details or editing screens are on the right.

11. App Store Requirements

Overlay 15: App Icon and Preview Screens

  • App Icon Design: Simple, recognizable, and representative of the app's functionality.
  • App Preview Screens: Show key features, with captions, optimized for the dimensions required by the App Store and Google Play Store.
